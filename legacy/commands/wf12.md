# /wf12 - 3 Cops Workflow (Quality Focus)

**Version 12**: 3 specialized adversarial reviewers (cops) at code review gate.

**Tag:** `[3-cops | 2-gate | 0-1 H]` — Simplicity + Coherence + Coverage

**Usage:**
- `/wf12 <task>` — Fully autonomous
- `/wf12 -h <task>` — Human gate after TDD_RED

**Research-backed**: 3-4 agents is optimal (Google Research). Each cop has distinct checklist, not just persona label.

---

## The 3 Cops

| Cop | Covers | Default | Key Rules |
|-----|--------|---------|-----------|
| `simplicity-cop` | Complexity + file bloat | REJECT | No speculative abstractions, <50 LOC = consolidate |
| `coherence-cop` | Pattern reuse + architecture | REJECT | Must search existing patterns, no layer violations |
| `coverage-cop` | Tests + regression | REJECT | Every function needs tests, edge cases required |

---

## Flag Detection

```bash
HUMAN_GATE=false
TASK="$ARGUMENTS"

if [[ "$ARGUMENTS" == *"-h"* ]]; then
  HUMAN_GATE=true
  TASK="${ARGUMENTS//-h/}"
  TASK="${TASK## }"
fi

echo "HUMAN_GATE: $HUMAN_GATE"
echo "TASK: $TASK"
```

## CRITICAL: NO PRE-EXISTING TEST FAILURES

**The baseline has 100% passing tests. ALWAYS.**

If tests fail, YOU broke them. Not "pre-existing." Fix the implementation.

## MANDATORY PROOF BLOCKS

### BASELINE_BLOCK (captured BEFORE any changes)

```text
┌─────────────────────────────────────────────┐
│ BASELINE_BLOCK                              │
├─────────────────────────────────────────────┤
│ Tests passing: [N]                          │
│ Tests failing: [N]                          │
│ Files in src/: [N]                          │
│ Git SHA: [hash]                             │
│ TIMESTAMP: [YYYY-MM-DD HH:MM:SS]            │
└─────────────────────────────────────────────┘
```

### EXECUTION_BLOCK (required for ANY completion claim)

```text
┌─────────────────────────────────────────────┐
│ EXECUTION_BLOCK                             │
├─────────────────────────────────────────────┤
│ $ [actual command run]                      │
│ [actual output - last 20+ lines]            │
│ EXIT_CODE: [0 or N]                         │
│ TIMESTAMP: [YYYY-MM-DD HH:MM:SS]            │
└─────────────────────────────────────────────┘
```

## Phase Flow (8 Phases, 2 Gates, 0-1 Human Gates)

### Phase 1: BASELINE_CAPTURE

Before touching ANY code:

1. Run existing tests, capture output
2. Count files in src/
3. Record git SHA
4. Output BASELINE_BLOCK

```bash
npm test 2>&1 | tee /tmp/baseline.txt || true
echo "FILES_IN_SRC: $(find src -type f 2>/dev/null | wc -l)"
echo "GIT_SHA: $(git rev-parse HEAD)"
```

**Do NOT proceed without BASELINE_BLOCK.**

---

### Phase 2: INTAKE (Autonomous)

Infer requirements from codebase and user task. NO questions asked.

1. Read the user's task/request
2. Explore relevant codebase files
3. **Search for existing patterns** (coherence-cop requirement)
4. Write spec to `.claude/temp/spec.md`
5. VERIFY: Show `cat .claude/temp/spec.md | head -30`

---

### Phase 3: PLANNING

Create architecture with **minimalism in mind**.

Write to `.claude/temp/architecture.md`:
- Component design (minimal files)
- **Reuse existing patterns** (list what exists)
- File structure (**justify each new file**)
- TDD test plan

**Minimalist gate**: If proposing >3 new files, justify each.

---

### Phase 4: GATE 1/2 - PLAN_REVIEW (Self-Review)

Review your own plan against the 3 cops' criteria:

```text
PLAN_SELF_REVIEW
----------------
Simplicity:    [OK|ISSUE] - Complexity + file count justified?
Coherence:     [OK|ISSUE] - Existing patterns reused? Layer violations?
Coverage:      [OK|ISSUE] - Test plan covers all requirements + edge cases?
----------------
VERDICT: [PROCEED|REVISE]
```

If REVISE: Fix and re-review (max 3).

---

### Phase 5: TDD_RED — Write Failing Tests

Spawn isolated coder:

```bash
claude -p "You are a TDD test writer with ZERO prior context.

Read: .claude/temp/spec.md, .claude/temp/architecture.md

Write RED tests:
- At least 1 test per requirement
- Tests MUST FAIL

After writing, run tests to verify they fail.
Output EXECUTION_BLOCK showing EXIT_CODE≠0." \
  --allowedTools Read,Write,Edit,Bash,Glob,Grep \
  --print
```

**Auto-stage tests:**
```bash
git add tests/
echo "✓ Tests staged after TDD_RED"
```

---

### HUMAN GATE (Only if `-h` flag)

**If HUMAN_GATE=true, STOP here and display:**

## [Architecture](.claude/temp/architecture.md)

[Summary of design + reused patterns + new files with justification]

| Test | Description |
|------|-------------|
| `[testName1]` | [What this test validates] |
| `[testName2]` | [What this test validates] |

Tests staged. Awaiting approval...

**Wait for user approval before proceeding to Phase 6.**

---

### Phase 6: TDD_GREEN — Implement

Spawn isolated coder:

```bash
claude -p "You are a TDD implementer with ZERO prior context.

Read: .claude/temp/spec.md, .claude/temp/architecture.md, test files

CRITICAL: Test files are READ-ONLY. Only edit src/ files.
CRITICAL: Prefer editing existing files over creating new ones.
CRITICAL: Reuse existing utilities/patterns.

Fix one error at a time until tests pass.

Output:
1. EXECUTION_BLOCK showing EXIT_CODE=0
2. REGRESSION_DELTA comparing to baseline" \
  --allowedTools Read,Write,Edit,Bash,Glob,Grep \
  --print
```

**VERIFY**: EXIT_CODE=0 and REGRESSION_DELTA=SAFE

---

### Phase 7: GATE 2/2 - CODE_REVIEW (3 Cops in PARALLEL)

**Spawn all 3 cops simultaneously:**

```bash
# Cop 1: Simplicity (complexity + file bloat)
claude -p "$(cat .claude/agents/simplicity-cop.md)

Review the code changes in this PR.
Read: src/ files, .claude/temp/spec.md

Output your SIMPLICITY_COP VERDICT." \
  --allowedTools Read,Glob,Grep,Bash \
  --print > /tmp/cop-simplicity.txt &

# Cop 2: Coherence (pattern reuse + architecture)
claude -p "$(cat .claude/agents/coherence-cop.md)

Review the code changes in this PR.
Read: src/ files, search for existing patterns, check imports.

Output your COHERENCE_COP VERDICT." \
  --allowedTools Read,Glob,Grep,Bash \
  --print > /tmp/cop-coherence.txt &

# Cop 3: Coverage (tests + regression)
claude -p "$(cat .claude/agents/coverage-cop.md)

Review the code changes in this PR.
Read: src/ files, tests/ files, check coverage.

Output your COVERAGE_COP VERDICT." \
  --allowedTools Read,Glob,Grep,Bash \
  --print > /tmp/cop-coverage.txt &

wait
cat /tmp/cop-simplicity.txt /tmp/cop-coherence.txt /tmp/cop-coverage.txt
```

#### Aggregate Verdicts

```text
3-COPS GATE SUMMARY
-------------------
Simplicity:    [PASS|REJECT]
Coherence:     [PASS|REJECT]
Coverage:      [PASS|REJECT]
-------------------
OVERALL: [ALL PASS|NEEDS_WORK]
```

**If ANY cop returns REJECT**: Fix issues and re-run that cop (max 3 per cop).

#### SMOKE_TEST

```bash
npm test 2>&1 | tee /tmp/smoke.txt
echo "EXIT_CODE: $?"
```

All must pass: 3 cops PASS + SMOKE_TEST PASS

---

### Phase 8: COMPLETION + AUTO_COMMIT

Only proceed if ALL pass:
- BASELINE_BLOCK captured
- EXIT_CODE=0
- REGRESSION_DELTA=SAFE
- All 3 cops PASS

```bash
git add -A
TASK_SUMMARY=$(head -5 .claude/temp/spec.md | grep -v "^#" | head -1)

git commit -m "$(cat <<EOF
feat: ${TASK_SUMMARY}

- Reviews: 3-cops (simplicity, coherence, coverage)
- All cops: PASS
- Regression: SAFE

Co-Authored-By: Claude <noreply@anthropic.com>
EOF
)"

echo "✓ Auto-committed"
git log -1 --oneline
```

**File delta check:**
```bash
echo "FILES BEFORE: $(cat /tmp/baseline.txt | grep FILES_IN_SRC | cut -d: -f2)"
echo "FILES AFTER: $(find src -type f 2>/dev/null | wc -l)"
```

Output: `✓ ORCHESTRATION COMPLETE (wf12)`

## Circuit Breakers

| Trigger | Action |
|---------|--------|
| No BASELINE_BLOCK | STOP |
| REGRESSION_DELTA = REGRESSION | STOP |
| Same cop fails 3x | STOP |
| Tests fail 5x | STOP |
| "Pre-existing failures" claim | RETRACT. Fix the code. |
| File count +5 without justification | STOP, consolidate |
